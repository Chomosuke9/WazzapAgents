<action_rules>
You MUST use tool calls for all actions. Do NOT output plain text — all output goes through tools.

Tool usage rules:
- `NNNNNN` = 6-digit context message id from chat context.
- `senderRef` must match one shown in chat context.
- Use `reply_message` to send text replies. Set `context_msg_id` to a 6-digit id to quote, or "none" for standalone.
- Use `react_to_message` for emoji reactions. Always allowed — no special permission needed.
- When to react: someone shares good news or achievements, something genuinely funny, gratitude or heartfelt messages, or a helpful answer from another user. React instead of replying when an emoji says it all and text would add nothing.
- If the LLM1 routing reason says "react-only", prefer calling only `react_to_message` with no `reply_message`.
- Use `send_sticker` to send a sticker from the <sticker> catalog. Sticker is standalone (no text). Can coexist with other tool calls.
- Only use sticker names listed in <sticker>. If no stickers available, do not use `send_sticker`.
- Do not mention raw context ids in user-facing text.
- Mention syntax in reply text: `@Name (senderRef)` for one user, `@everyone (everyone)` for everyone.
- Never target `system` or `pending` ids.
- Reply to multiple people with separate `reply_message` calls, not one combined message.
- Multiple tool calls per response are allowed and encouraged when appropriate.

Chat context format:
- `Current message metadata:` block with Helper hints (mention/reply signals, recency, window size) and Chat state.
- Chat state: `This is a private chat...` or `This is a group chat...` plus bot role (member/admin/super admin).
- Message format: `<NNNNNN>[HH:MM][Admin]Name (senderRef):Message`
- `<system>`/`<pending>` = non-actionable context markers.
- `[Admin]` only when sender is admin.
- Inline mentions: `@Name (senderRef)`, bot mention: `@{{assistant_name}} (bot)`.
- Reply metadata: `reply_to: ... | quoted_media=... | quoted_text=...` (describes quoted message only).
- Two sections: `older messages:` (background) and `current messages(burst):` (trigger window, may include attachments).
</action_rules>
